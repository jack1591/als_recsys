# Instacart ALS Recommender System

Рекомендательная система на основе коллаборативной фильтрации, построенная на [датасете Instacart Market Basket](https://www.kaggle.com/competitions/instacart-market-basket-analysis/data) с использованием метода Alternating Least Squares (ALS) и REST API на FastAPI.

## Описание

Система обучает латентные эмбеддинги пользователей и товаров на истории покупок с использованием неявной обратной связи. BM25-взвешивание применяется перед обучением для снижения смещения в сторону популярных товаров. Обученная модель обслуживается через REST API, возвращая персональные рекомендации по ID пользователя.

## Быстрый старт

### 1. Скачать данные

Скачайте датасет Instacart с [Kaggle](https://www.kaggle.com/competitions/instacart-market-basket-analysis/data) и поместите CSV-файлы в `data/raw/`:

```
data/raw/
├── orders.csv
├── order_products__prior.csv
├── products.csv
├── aisles.csv
└── departments.csv
```

### 2. Подготовить данные

```bash
python prepare_data.py
```

Фильтрует пользователей с 5–20 покупками, отбирает топ-1500 товаров, строит разреженную матрицу взаимодействий и сохраняет разбивку 80/20 в `data/processed/`.

### 3. Обучить модель

```bash
python main.py
```

Обучает ALS на BM25-взвешенных взаимодействиях, оценивает качество на тестовой выборке и сохраняет модель в `model/als_model.pkl`.

### 4. Запустить API

```bash
uvicorn api:app --reload --port 8000
```

## API

#### `GET /recommend/{user_id}?k=10`

Возвращает топ-k персональных рекомендаций для пользователя.

**Параметры:**

| Параметр | Тип | По умолчанию | Описание |
|----------|-----|--------------|----------|
| `user_id` | int | — | ID пользователя из датасета Instacart |
| `k` | int | 10 | Количество рекомендаций (1–50) |

**Пример запроса:**
```bash
curl http://localhost:8000/recommend/1?k=5
```

**Пример ответа:**
```json
{
  "user_id": 1,
  "recommendations": [
    {
      "product_id": 24852,
      "product_name": "Banana",
      "aisle": "fresh fruits",
      "department": "produce",
      "score": 0.94312
    }
  ]
}
```

#### `GET /health`

Возвращает `{"status": "ok"}` — используется для проверки работоспособности сервиса.

#### `GET /docs`

Swagger UI с интерактивной документацией (генерируется FastAPI автоматически).

## Параметры модели

| Параметр | Значение |
|----------|----------|
| Алгоритм | Alternating Least Squares (implicit) |
| Взвешивание | BM25 (K1=50, B=0.8) |
| Факторы | 64 |
| Регуляризация | 0.01 |
| Итерации | 20 |
| Разбивка | 80/20 по товарам каждого пользователя |

## Метрики качества

Оцениваются на отложенных 20% покупок каждого пользователя:

| Метрика | Описание |
|---------|----------|
| NDCG@5, NDCG@10 | Основная метрика качества ранжирования |
| Precision@5, Precision@10 | Доля релевантных товаров среди рекомендованных |
| Recall@5, Recall@10 | Доля найденных релевантных товаров |
| Coverage | Доля каталога, которая когда-либо рекомендовалась |

## Примечания по обработке данных

- Источник: `order_products__prior.csv` (полная история покупок, ~32M строк)
- Отбираются пользователи с 5–20 купленными товарами
- Выбираются топ-1500 товаров по частоте покупок
- Разбивка train/test производится по каждому пользователю: 20% товаров уходят в тест
- Пользователи с единственным товаром целиком попадают в обучающую выборку
